🏗 Swin Transformer 아키텍처

Swin Transformer는 시프트 윈도우 메커니즘으로 계층적 특징 맵을 생성하는 비전 트랜스포머입니다. 이미지를 4×4 패치로 분할한 후, 4개의 스테이지를 거치며 해상도는 줄이고 채널은 늘려갑니다.

💡 왜 Swin Transformer인가?

ViT의 한계: ViT는 모든 패치 쌍에 대해 전역 어텐션을 계산하므로 이미지 크기에 대해 O(n²)의 연산량이 필요합니다. 고해상도 이미지에 비실용적입니다.

Swin의 해결책: 윈도우 어텐션으로 O(n) 선형 복잡도를 달성하면서도, 시프트 윈도우로 윈도우 간 정보 교환을 보장합니다.

계층적 구조: CNN처럼 다양한 해상도의 특징 피라미드를 생성하여 객체 탐지, 세그멘테이션 등 다양한 다운스트림 태스크에 활용됩니다.

📐 입력

224×224 RGB 이미지를 4×4 패치로 분할 → 56×56 = 3136개 토큰 (C=96)

🔄 4 Stages

각 Stage에서 W-MSA / SW-MSA 교대 수행. Stage 전환 시 Patch Merging으로 해상도 ½ 축소

🎯 출력

최종 7×7 특징 맵 → Global Average Pooling → 분류 헤드 (또는 FPN으로 밀집 예측)

▦ 패치 파티션 & 패치 머징

패치 파티션은 이미지를 겹치지 않는 4×4 픽셀 패치로 분할하여 토큰 시퀀스를 생성합니다. 패치 머징은 Stage 전환 시 2×2 인접 패치를 결합하여 해상도를 절반으로 줄이고 채널을 2배로 늘립니다.

💡 왜 패치 머징인가?

계층적 표현: CNN의 풀링처럼 점진적으로 해상도를 줄이면서 더 넓은 수용 영역(receptive field)을 확보합니다.

연산량 절감: 해상도가 절반이 되면 토큰 수가 ¼로 줄어 이후 레이어의 어텐션 비용이 크게 감소합니다.

다중 스케일: 각 Stage의 특징 맵이 서로 다른 해상도를 가지므로 FPN 등에 직접 활용할 수 있습니다.

🐍 PyTorch 코드 
# Patch Partition (Stage 1 입력)
import torch.nn as nn

class PatchEmbed(nn.Module):
    def __init__(self, patch_size=4, in_chans=3, embed_dim=96):
        super().__init__()
        self.proj = nn.Conv2d(
            in_chans, embed_dim,
            kernel_size=patch_size,
            stride=patch_size       # non-overlapping
        )
        self.norm = nn.LayerNorm(embed_dim)

    def forward(self, x):
        # x: [B, 3, 224, 224]
        x = self.proj(x)           # [B, 96, 56, 56]
        x = x.flatten(2).transpose(1, 2)  # [B, 3136, 96]
        return self.norm(x)

# Patch Merging (Stage 전환)
class PatchMerging(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.reduction = nn.Linear(4 * dim, 2 * dim, bias=False)
        self.norm = nn.LayerNorm(4 * dim)

    def forward(self, x, H, W):
        # x: [B, H*W, C]
        x = x.view(-1, H, W, x.shape[-1])
        # 2×2 인접 패치를 concat
        x0 = x[:, 0::2, 0::2, :]  # 좌상
        x1 = x[:, 1::2, 0::2, :]  # 좌하
        x2 = x[:, 0::2, 1::2, :]  # 우상
        x3 = x[:, 1::2, 1::2, :]  # 우하
        x = torch.cat([x0, x1, x2, x3], -1)  # [B, H/2, W/2, 4C]
        x = self.norm(x.view(-1, (H//2)*(W//2), 4*x.shape[-1]))
        return self.reduction(x)   # [B, H/2*W/2, 2C]

⊞ Shifted Window Attention

W-MSA는 각 윈도우(M×M) 내에서만 셀프 어텐션을 수행합니다. 다음 레이어에서는 윈도우를 (M/2, M/2)만큼 이동시킨 SW-MSA를 적용하여 윈도우 간 정보를 교환합니다.

💡 왜 Shifted Window인가?

윈도우 어텐션의 한계: 고정 윈도우(W-MSA)만 사용하면 윈도우 경계에 있는 패치끼리 정보를 주고받을 수 없습니다.

시프트의 효과: 윈도우를 반 칸 이동하면 기존 윈도우 경계에 걸쳐있던 패치들이 같은 윈도우에 포함되어 교차 윈도우 연결(cross-window connection)이 생성됩니다.

효율적 구현: 순환 시프트와 어텐션 마스크로 추가 패딩 없이 배치(batch) 연산이 가능합니다.

🐍 PyTorch 코드 
import torch
import torch.nn.functional as F

def window_partition(x, window_size):
    """x: [B, H, W, C] → [num_windows*B, M, M, C]"""
    B, H, W, C = x.shape
    M = window_size
    x = x.view(B, H // M, M, W // M, M, C)
    windows = x.permute(0, 1, 3, 2, 4, 5).contiguous()
    return windows.view(-1, M, M, C)

def window_reverse(windows, window_size, H, W):
    """[num_windows*B, M, M, C] → [B, H, W, C]"""
    M = window_size
    B = int(windows.shape[0] / (H * W / M / M))
    x = windows.view(B, H // M, W // M, M, M, -1)
    return x.permute(0, 1, 3, 2, 4, 5).contiguous().view(B, H, W, -1)

# Cyclic Shift (SW-MSA)
shift_size = window_size // 2
shifted_x = torch.roll(x, shifts=(-shift_size, -shift_size), dims=(1, 2))
# → shifted window partition → attention with mask → reverse → roll back

🪟 W-MSA

고정된 M×M 윈도우 내에서만 어텐션. 윈도우 간 정보 교환 없음. 연산량: O(n·M²)

↗ SW-MSA

윈도우를 (M/2, M/2) 이동. 기존 경계 패치들이 새 윈도우에서 만남. 교차 연결 생성

🔄 교대 적용

홀수 레이어 = W-MSA, 짝수 레이어 = SW-MSA. 모든 패치가 점진적으로 연결됨

📐 상대적 위치 바이어스 (Relative Position Bias)

Swin Transformer는 절대 위치 인코딩 대신 상대적 위치 바이어스 B를 어텐션 스코어에 직접 더합니다. 두 토큰 간의 상대적 거리에 따라 학습 가능한 바이어스 값이 결정됩니다.

💡 왜 상대적 위치 바이어스인가?

절대 위치의 한계: ViT의 절대 위치 임베딩은 입력 해상도가 변하면 보간(interpolation)이 필요하며, 패치 간 상대적 관계를 직접적으로 인코딩하지 못합니다.

상대적 바이어스의 장점: 두 패치 사이의 거리와 방향만으로 위치를 표현하므로, 윈도우 크기가 달라져도 자연스럽게 적용됩니다. 논문 실험에서 절대 위치 대비 +1.2% top-1 정확도 향상.

구현: (2M-1)×(2M-1) 크기의 학습 가능한 테이블에서 상대 좌표 인덱스로 바이어스 값을 조회합니다.

🐍 PyTorch 코드 
import torch
import torch.nn as nn

M = 7  # 윈도우 크기

# 학습 가능한 바이어스 테이블: (2M-1) * (2M-1) 크기
relative_position_bias_table = nn.Parameter(
    torch.zeros((2 * M - 1) * (2 * M - 1), num_heads)
)
nn.init.trunc_normal_(relative_position_bias_table, std=0.02)

# 상대 좌표 인덱스 계산
coords_h = torch.arange(M)
coords_w = torch.arange(M)
coords = torch.stack(torch.meshgrid([coords_h, coords_w], indexing='ij'))
coords_flatten = torch.flatten(coords, 1)   # [2, M*M]

# 상대 좌표: [2, M², M²]
relative_coords = coords_flatten[:, :, None] - coords_flatten[:, None, :]
relative_coords = relative_coords.permute(1, 2, 0).contiguous()
relative_coords[:, :, 0] += M - 1  # 0부터 시작하도록 오프셋
relative_coords[:, :, 1] += M - 1
relative_coords[:, :, 0] *= 2 * M - 1
relative_position_index = relative_coords.sum(-1)  # [M², M²]

# 어텐션에 바이어스 적용
# attn: softmax(QK^T / sqrt(d) + B)
B = relative_position_bias_table[relative_position_index.view(-1)].view(M*M, M*M, -1)
attn = attn + B.permute(2, 0, 1).unsqueeze(0)
