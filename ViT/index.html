전체 아키텍처

Vision Transformer(ViT)는 이미지를 패치로 분할한 후, Transformer 인코더를 적용하여 분류하는 모델입니다.

💡 왜 Vision Transformer인가?

CNN의 한계: 귀납적 편향(locality, translation equivariance)이 내장되어 있어 적은 데이터에선 유리하지만, 수용야(receptive field)가 제한적입니다. 먼 영역 간 관계를 파악하려면 많은 층을 쌓아야 합니다.

ViT의 접근: Self-Attention으로 모든 패치 간 관계를 한 번에 직접 모델링합니다. 대규모 데이터(JFT-300M 등)에서 사전학습하면 CNN을 능가하는 성능을 달성합니다.

🐍 PyTorch 코드 
import timm

# ViT-Base/16: 패치 16×16, 12레이어, 12헤드, d=768
model = timm.create_model(
    'vit_base_patch16_224',
    pretrained=True,
    num_classes=1000
)
# 입력: [batch, 3, 224, 224]
# 출력: [batch, 1000]
output = model(images)

핵심 아이디어

이미지를 고정 크기 패치로 나누고, 각 패치를 NLP의 "토큰"처럼 취급하여 Transformer에 입력합니다.

vs CNN

CNN은 지역적 특징을 계층적으로 추출하지만, ViT는 Self-Attention으로 모든 패치 간 전역적 관계를 한 번에 파악합니다.

주요 설정

ViT-Base: 패치 16×16, 12 레이어, 12 헤드, d=768
입력: 224×224 이미지 → 196개 패치 + 1 CLS 토큰

이미지 패치 분할

입력 이미지를 P×P 크기의 고정 패치로 분할합니다. 224×224 이미지를 16×16 패치로 나누면 14×14 = 196개의 패치가 생성됩니다.

💡 왜 패치로 나누는가?

픽셀 단위의 한계: 224×224 이미지는 50,176개 픽셀입니다. Self-Attention의 연산량은 시퀀스 길이의 제곱(O(n²))이므로, 픽셀 단위로 처리하면 계산이 불가능합니다.

패치 → 토큰: 16×16 패치로 나누면 시퀀스 길이가 196으로 줄어 Transformer에 실용적으로 입력할 수 있습니다. 각 패치는 NLP에서의 단어 토큰에 해당합니다.

🐍 PyTorch 코드 
import torch.nn as nn

P = 16  # 패치 크기
# Conv2d를 패치 추출기로 사용 (stride=P)
patch_embed = nn.Conv2d(
    in_channels=3, out_channels=768,
    kernel_size=P, stride=P
)
# 입력: [1, 3, 224, 224]
# 출력: [1, 768, 14, 14] → flatten → [1, 196, 768]
patches = patch_embed(image).flatten(2).transpose(1, 2)

원본 이미지

패치 분할 결과 ?개 패치

패치 → 1D 시퀀스 변환 (Flatten)

패치 임베딩 (Linear Projection)

각 패치를 1D 벡터로 펼친 후 학습 가능한 선형 레이어(E)를 통해 D차원 임베딩으로 변환합니다.

💡 왜 선형 투영인가?

차원 맞춤: 원시 패치 픽셀(P²×C = 768차원)을 Transformer가 처리할 수 있는 모델 차원 D로 투영합니다. 이는 NLP에서 단어를 임베딩 벡터로 변환하는 것과 동일한 역할입니다.

CNN 특징추출 대체: CNN은 여러 층의 합성곱으로 특징을 추출하지만, ViT는 단일 선형 변환으로 패치를 표현 공간에 사상합니다. 단순하지만 대규모 데이터에서 효과적입니다.

🐍 PyTorch 코드 
import torch.nn as nn

P = 16   # 패치 크기
C = 3    # 채널 수 (RGB)
D = 768  # 임베딩 차원

# 방법 1: Linear Projection
proj = nn.Linear(P * P * C, D)
# [batch, N, P²·C] → [batch, N, D]

# 방법 2: Conv2d (동등하지만 더 효율적)
proj = nn.Conv2d(C, D, kernel_size=P, stride=P)
# [batch, 3, 224, 224] → [batch, D, 14, 14]

Flatten + Linear Projection

왜 선형 변환?

각 패치의 픽셀 값(P²×C)을 모델 차원 D로 투영합니다. 이는 NLP에서 단어를 임베딩 벡터로 변환하는 것과 동일한 역할입니다.

차원 계산

16×16 패치, RGB(3채널): 16×16×3 = 768차원
→ 선형변환 → D=768차원 (ViT-Base)

위치 임베딩 (Position Embedding)

패치의 공간적 위치 정보를 보존하기 위해 학습 가능한 위치 임베딩을 추가합니다.

💡 왜 위치 임베딩이 필요한가?

Transformer는 순서를 모릅니다: Self-Attention은 집합(set) 연산이라 토큰의 순서를 구분하지 못합니다. 패치를 뒤섞어도 결과가 동일합니다.

공간 정보 주입: 위치 임베딩을 더하면 모델이 "이 패치는 왼쪽 위" "저 패치는 오른쪽 아래"를 알 수 있습니다. 학습 후 가까운 위치끼리 유사한 임베딩을 가지게 됩니다.

🐍 PyTorch 코드 
import torch
import torch.nn as nn

N = 196  # 패치 수 (14×14)
D = 768  # 임베딩 차원

# 학습 가능한 위치 임베딩 (CLS 포함 → N+1)
pos_embed = nn.Parameter(
    torch.randn(1, N + 1, D) * 0.02
)
# 패치 임베딩 + 위치 임베딩
z_0 = patch_embeddings + pos_embed

위치 임베딩 히트맵

위치 간 코사인 유사도

학습 가능 임베딩

Transformer와 달리 ViT는 고정된 Sin/Cos 대신 학습 가능한(Learnable) 위치 임베딩을 사용합니다. 학습 후 가까운 패치끼리 유사한 패턴을 보입니다.

2D 구조 보존

학습된 위치 임베딩은 2D 공간적 구조를 자연스럽게 학습합니다. 인접한 패치 위치의 임베딩이 높은 유사도를 가집니다.

CLS 토큰

BERT에서 영감을 받아 학습 가능한 [CLS] 토큰을 패치 시퀀스 앞에 추가합니다. 이 토큰이 이미지 전체의 표현을 학습합니다.

💡 왜 CLS 토큰을 사용하는가?

전역 대표 벡터: Self-Attention을 거치며 CLS 토큰은 모든 패치의 정보를 전역적으로 집약합니다. 이 토큰만으로 이미지 전체를 대표할 수 있습니다.

대안 — Global Average Pooling: 모든 패치 출력을 평균하는 방법(GAP)도 가능하지만, 원 논문은 BERT의 [CLS] 방식을 따릅니다. 성능 차이는 미미하나, CLS 방식이 Transformer 전통에 일관됩니다.

🐍 PyTorch 코드 
import torch
import torch.nn as nn

D = 768  # 임베딩 차원
# 학습 가능한 CLS 토큰
cls_token = nn.Parameter(torch.randn(1, 1, D))

# 배치 크기만큼 확장 후 시퀀스 앞에 추가
cls_tokens = cls_token.expand(batch_size, -1, -1)
# [batch, 1, D] + [batch, N, D] → [batch, N+1, D]
z = torch.cat([cls_tokens, patch_embeddings], dim=1)

CLS 토큰 + 패치 임베딩 시퀀스

CLS 토큰의 역할

Self-Attention을 통해 모든 패치 정보를 전역적으로 집약합니다. 최종 분류 시 CLS 토큰의 출력만 사용합니다.

시퀀스 구성

[CLS, Patch₁, Patch₂, …, Patch_N]
총 N+1개 토큰 (196패치 + 1 CLS = 197)

Transformer 인코더

ViT의 핵심 구조입니다. Layer Norm → Multi-Head Self-Attention → 잔차 연결 → Layer Norm → MLP → 잔차 연결의 구조가 L번 반복됩니다.

💡 왜 여러 층을 쌓는가?

계층적 특징 학습: 초기 레이어는 가장자리· 텍스처 같은 저수준 패턴을, 후반 레이어는 객체·장면 같은 고수준 의미를 학습합니다. 층을 쌓을수록 추상화 수준이 높아집니다.

Pre-Norm: ViT는 원래 Transformer(Post-Norm)와 달리 LayerNorm을 Attention/MLP 앞에 적용합니다. 이로써 학습이 더 안정적으로 수렴합니다.

🐍 PyTorch 코드 
import torch.nn as nn

D = 768        # 임베딩 차원
num_heads = 12 # 어텐션 헤드 수
L = 12         # 인코더 레이어 수

encoder_layer = nn.TransformerEncoderLayer(
    d_model=D,
    nhead=num_heads,
    dim_feedforward=3072,  # 4 × D
    activation='gelu',
    norm_first=True  # Pre-Norm (ViT 스타일)
)
encoder = nn.TransformerEncoder(
    encoder_layer, num_layers=L
)

Pre-Norm vs Post-Norm

ViT는 원래 Transformer와 달리 Pre-Norm을 사용합니다. LayerNorm을 Attention/MLP 앞에 적용하여 학습 안정성을 높입니다.

수식


Self-Attention 시각화

Self-Attention은 ViT의 핵심입니다. 각 패치 토큰이 다른 모든 패치에 대해 "얼마나 관련 있는가?"를 계산하고, 그 관련도에 따라 정보를 가중 합산합니다. CNN과 달리 멀리 떨어진 패치끼리도 한 번에 직접 정보를 교환할 수 있는 것이 핵심 장점입니다.

💡 왜 Self-Attention인가?

전역 수용야: CNN의 합성곱 커널은 3×3 등 작은 영역만 봅니다. 먼 영역을 연결하려면 많은 층이 필요합니다. Self-Attention은 모든 패치 쌍의 관계를 한 번의 연산으로 직접 모델링합니다.

√d_k 스케일링: Q·KT 내적 값이 차원 d_k에 비례하여 커지면 softmax가 포화됩니다. √d_k로 나누어 안정적인 기울기를 유지합니다.

🐍 PyTorch 코드 
import torch.nn as nn

D = 768       # 임베딩 차원
num_heads = 6 # 어텐션 헤드 수

mha = nn.MultiheadAttention(
    embed_dim=D,
    num_heads=num_heads,
    batch_first=True
)
# Q, K, V 모두 같은 입력 (Self-Attention)
# 입력: [batch, N+1, D]
attn_output, attn_weights = mha(z, z, z)

❓ Query (Q) — 질문

현재 토큰이 "나는 어떤 정보가 필요한가?"를 표현하는 벡터입니다. 슬라이더로 선택한 패치(또는 CLS)의 Q가 다른 모든 토큰의 K와 내적되어 관련도 점수를 만듭니다.

🔑 Key (K) — 열쇠

각 토큰이 "나는 이런 정보를 가지고 있다"고 알리는 벡터입니다. Q·KT 내적 값이 높을수록 두 토큰의 관계가 강하고, 어텐션 점수(막대 높이)가 높게 나타납니다.

💡 Value (V) — 내용

실제로 전달할 정보를 담은 벡터입니다. Softmax로 정규화된 어텐션 점수를 가중치로 삼아 모든 토큰의 V를 합산한 결과가 이 토큰의 새로운 표현이 됩니다.

🎯 Multi-Head — 다중 관점

헤드를 여럿 두면 각 헤드가 독립적으로 서로 다른 관계 패턴을 학습합니다. 아래 시뮬레이션에서 헤드 1–2는 지역적, 3–4는 전역적, 5–6은 같은 행(가로) 패턴을 나타냅니다.

어텐션 맵 (막대 그래프)

선택 패치(Query)가 각 토큰(Key)에 보내는 어텐션 가중치. 막대가 높을수록 해당 토큰에 더 많이 주목합니다.

이미지 위 어텐션 오버레이

노란색이 밝을수록 선택 패치가 해당 위치에 많은 주의를 기울임. 빨간 테두리가 현재 선택된 Query 패치입니다.

어텐션 가중치 행렬 (전체 N × N)

행 = Query 토큰, 열 = Key 토큰. 밝은 셀일수록 해당 행의 토큰이 해당 열 토큰을 강하게 참조합니다. 대각선이 밝으면 자기 자신을 많이 참조하는 패턴입니다.

분류 헤드 (MLP Head)

Transformer 인코더의 출력 중 CLS 토큰 벡터만 추출하여 MLP Head를 통해 최종 클래스를 예측합니다.

💡 왜 CLS 토큰만 사용하는가?

정보 집약: L개 인코더 레이어의 Self-Attention을 거치며 CLS 토큰은 모든 패치의 정보를 전역적으로 흡수합니다. 마지막 레이어의 CLS 출력은 이미지 전체를 요약한 벡터입니다.

Pre-training vs Fine-tuning: 사전학습 시에는 은닉층 + tanh 활성화를 포함한 MLP Head를 사용하고, 미세조정 시에는 단일 Linear 레이어로 교체하여 ImageNet 등의 클래스 수에 맞춥니다.

🐍 PyTorch 코드 
import torch.nn as nn

D = 768           # 임베딩 차원
num_classes = 1000  # ImageNet 클래스 수

# Fine-tuning: 단순 Linear Head
head = nn.Sequential(
    nn.LayerNorm(D),
    nn.Linear(D, num_classes)
)
# CLS 토큰 출력 → 분류
# z_L: [batch, N+1, D]
cls_output = z_L[:, 0]  # [batch, D]
logits = head(cls_output)  # [batch, 1000]

Pre-training

사전학습 시: 은닉층 + tanh 활성화 포함 MLP Head
(Linear → tanh → Linear)

Fine-tuning

미세조정 시: 단일 Linear Layer로 교체
D차원 → 클래스 수(예: 1000 for ImageNet)
