전체 아키텍처

Transformer 모델의 인코더-디코더 구조입니다. 각 블록을 클릭하면 상세 설명으로 이동합니다.

💡 왜 Transformer인가?

RNN의 한계: 순차 처리(sequential)라 병렬화 불가능하고, 긴 시퀀스에서 기울기 소실 문제가 심합니다. 문장이 길어질수록 앞쪽 단어의 정보가 사라집니다.

CNN의 한계: 고정된 수용야(receptive field)로 장거리 의존성을 직접 포착할 수 없고, 여러 레이어를 쌓아야 합니다.

Transformer의 해결: Self-Attention으로 모든 위치 쌍을 O(1) 거리로 직접 연결하고, 완전한 병렬 처리가 가능합니다. 이것이 "Attention Is All You Need"의 핵심입니다.

🐍 PyTorch 코드 
import torch.nn as nn

transformer = nn.Transformer(
    d_model=512, nhead=8,
    num_encoder_layers=2,
    num_decoder_layers=2,
    dim_feedforward=2048
)
# src: [seq_len, batch, 512]
# tgt: [seq_len, batch, 512]
out = transformer(src, tgt)

입력 임베딩 & 위치 인코딩

토큰을 벡터로 변환하고 위치 인코딩을 더합니다.

💡 왜 위치 정보가 필요한가?

Self-Attention은 순서를 모릅니다: 어텐션 연산은 집합(set) 연산이라 입력 순서를 구분하지 못합니다. 위치 인코딩 없으면 "나는 너를 사랑한다"와 "너는 나를 사랑한다"가 동일하게 처리됩니다.

Sin/Cos 인코딩의 장점: 학습 없이 무한한 길이의 위치를 표현할 수 있고, 상대적 위치 관계가 내적(dot-product)으로 자연스럽게 인코딩됩니다.

🐍 PyTorch 코드 
import torch
import math

def positional_encoding(max_len, d_model):
    pe = torch.zeros(max_len, d_model)
    pos = torch.arange(0, max_len).unsqueeze(1).float()
    div = torch.exp(
        torch.arange(0, d_model, 2).float()
        * -(math.log(10000.0) / d_model)
    )
    pe[:, 0::2] = torch.sin(pos * div)
    pe[:, 1::2] = torch.cos(pos * div)
    return pe  # [max_len, d_model]

Scaled Dot-Product Attention

💡 왜 Self-Attention인가?

RNN: 토큰 간 거리 O(n) — 10번째 단어가 1번째 단어를 참조하려면 9단계를 거칩니다. 정보가 점점 희석됩니다.

Self-Attention: 모든 토큰 쌍의 거리 O(1) — Query와 Key의 내적으로 직접 연결합니다.

Scaling (√dk): 차원이 클수록 내적 값이 커져서 softmax가 포화됩니다. √dk로 나누면 기울기가 적절한 범위에 유지됩니다.

🐍 PyTorch 코드 
import torch
import torch.nn.functional as F
import math

def scaled_dot_product_attention(Q, K, V):
    d_k = Q.size(-1)
    # QK^T / sqrt(d_k)
    scores = torch.matmul(Q, K.transpose(-2, -1))
    scores = scores / math.sqrt(d_k)
    # softmax → 가중치
    weights = F.softmax(scores, dim=-1)
    # 가중합
    return torch.matmul(weights, V)
단계 1/5: Q, K, V 입력

멀티-헤드 어텐션

멀티-헤드: 입력을 여러 헤드로 분할하여 병렬로 어텐션을 수행합니다.

💡 왜 여러 헤드가 필요한가?

단일 헤드의 한계: 하나의 어텐션은 하나의 "관점"만 포착합니다. 예를 들어 구문 관계만 학습하면 의미 관계는 놓칩니다.

멀티-헤드의 장점: 각 헤드가 서로 다른 부분공간(subspace)에서 독립적으로 어텐션을 계산합니다. Head 1은 구문, Head 2는 지시어, Head 3는 의미 유사성 등 다양한 관계를 병렬 학습합니다.

계산 비용: dmodel을 h개로 나누므로 (dk = dmodel/h), 총 연산량은 단일 헤드와 동일합니다.

🐍 PyTorch 코드 
import torch.nn as nn

mha = nn.MultiheadAttention(
    embed_dim=512, num_heads=2
)
# query, key, value: [seq_len, batch, 512]
attn_out, attn_weights = mha(query, key, value)
# attn_out: [seq_len, batch, 512]

Add & Norm (잔차 연결)

Skip connection으로 입력을 보존하고, LayerNorm으로 정규화합니다.

💡 왜 잔차 연결 + 정규화인가?

기울기 소실 문제: 레이어가 깊어지면 역전파 시 기울기가 점점 작아져 앞쪽 레이어가 학습되지 않습니다.

잔차 연결(ResNet 아이디어): 기울기가 skip connection을 통해 직통 경로로 전달되어, 6~12개 레이어를 쌓아도 안정적으로 학습할 수 있습니다.

LayerNorm: 매 단계마다 값의 분포를 "평균=0, 분산=1" 근처로 맞춰 학습이 빠르고 안정적으로 진행됩니다. BatchNorm과 달리 시퀀스 길이에 무관합니다.

🐍 PyTorch 코드 
import torch.nn as nn

layer_norm = nn.LayerNorm(d_model)  # d_model=512

# Add & Norm
# x: 서브레이어 입력, sublayer_out: 어텐션/FFN 출력
out = layer_norm(x + sublayer_out)

피드 포워드 네트워크 (FFN)

💡 왜 위치별 FFN인가?

어텐션의 한계: Self-Attention은 본질적으로 "가중 평균" — 선형 연산에 가깝습니다. 비선형성 없이는 레이어를 아무리 쌓아도 하나의 어텐션과 같습니다.

FFN의 역할: 각 토큰의 표현을 독립적으로 비선형 변환합니다. dmodel(512) → dff(2048) → dmodel(512)로 확장했다 축소하여, 넓은 공간에서 복잡한 패턴을 포착합니다.

"Position-wise": 같은 FFN이 모든 위치에 동일하게 적용됩니다. 토큰 간 정보 교환은 오직 어텐션에서만 일어납니다.

🐍 PyTorch 코드 
import torch.nn as nn

ffn = nn.Sequential(
    nn.Linear(512, 2048),   # d_model → d_ff
    nn.ReLU(),
    nn.Linear(2048, 512)    # d_ff → d_model
)
# 입력값: 1.0
# x: [seq_len, batch, 512]
out = ffn(x)

디코더 특수 구조

Masked Self-Attention과 Cross-Attention의 차이를 이해합니다.

💡 왜 Autoregressive 디코딩인가?

문제: 번역/생성 시 출력 토큰은 이전에 생성된 토큰에 의존합니다. "나는"→"I"→"am" 순서로 생성해야 하며, "am"을 생성할 때 아직 나오지 않은 미래 토큰을 볼 수 없습니다.

Masked Self-Attention: 현재 위치 이후의 토큰을 −∞로 마스킹하여 softmax에서 0이 되게 합니다. 학습 시에도 미래 정보 유출(leaking)을 방지합니다.

Cross-Attention: 디코더의 Query가 인코더의 Key/Value를 참조합니다. 번역에서 "am"을 생성할 때 원문 "학생입니다"에 집중할 수 있습니다.

🐍 PyTorch 코드 
import torch.nn as nn

decoder_layer = nn.TransformerDecoderLayer(
    d_model=512, nhead=8,
    dim_feedforward=2048
)
decoder = nn.TransformerDecoder(
    decoder_layer, num_layers=6
)
# memory: 인코더 출력 [src_len, batch, 512]
# tgt: 디코더 입력 [tgt_len, batch, 512]
out = decoder(tgt, memory)
