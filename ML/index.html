머신러닝 전체 파이프라인

Machine Learning의 전체 워크플로우입니다. 각 블록을 클릭하면 해당 섹션으로 이동합니다.

💡 왜 머신러닝인가?

규칙 기반의 한계: 전통 프로그래밍은 모든 규칙을 사람이 직접 작성해야 합니다. 이미지 인식, 자연어 처리 같은 복잡한 패턴은 명시적 규칙으로 표현이 불가능합니다.

데이터로부터 학습: ML은 데이터에서 패턴을 자동으로 추출합니다. 데이터가 많을수록, 더 정확한 모델을 만들 수 있습니다.

선형 회귀 (Linear Regression)

회귀 분석의 가장 기본 형태로, 데이터에 가장 잘 맞는 직선을 찾습니다.

💡 왜 선형 회귀인가?

ML의 기초 중 기초: 모든 복잡한 모델(신경망, 트리 등)도 결국 손실 함수 최소화라는 같은 원리를 사용합니다. 선형 회귀는 이 원리를 가장 단순하게 보여줍니다.

해석 가능성: 기울기 w가 "입력이 1 증가할 때 출력의 변화량"을 직접 의미하므로, 모델의 동작을 완벽히 이해할 수 있습니다.

🐍 scikit-learn 코드 
from sklearn.linear_model import LinearRegression

model = LinearRegression()
model.fit(X_train, y_train)
# w = model.coef_[0], b = model.intercept_
predictions = model.predict(X_test)

경사 하강법 (Gradient Descent)

경사 하강법은 손실 함수의 기울기를 따라 파라미터를 업데이트하여 최적값을 찾는 알고리즘입니다.

💡 왜 경사 하강법인가?

닫힌 해가 없는 경우: 선형 회귀는 수학적 해(정규방정식)가 존재하지만, 대부분의 ML/DL 모델은 해석적 해가 없습니다. 반복적 최적화가 유일한 방법입니다.

확장성: 수백만 개의 파라미터도 그래디언트만 계산할 수 있으면 최적화할 수 있어, 딥러닝의 핵심 엔진입니다.

🐍 PyTorch 코드 
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

for epoch in range(100):
    loss = criterion(model(X), y)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

분류 (Classification)

분류는 입력 데이터를 이산적인 클래스로 나누는 문제입니다. 로지스틱 회귀는 가장 기본적인 이진 분류 모델입니다.

💡 왜 분류인가?

현실의 의사결정: 스팸/정상 메일, 양성/악성 종양, 정상/사기 거래 등 현실의 많은 문제가 분류 문제입니다.

확률적 해석: Sigmoid 함수를 통해 출력을 0~1 확률로 변환하므로, "얼마나 확신하는가"를 알 수 있습니다.

🐍 scikit-learn 코드 
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import confusion_matrix

model = LogisticRegression()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
cm = confusion_matrix(y_test, y_pred)

과적합과 정규화 (Overfitting & Regularization)

모델이 학습 데이터에 과적합되면 새로운 데이터에 일반화하지 못합니다. 정규화로 이를 방지합니다.

💡 왜 정규화인가?

복잡한 모델의 위험: 파라미터가 많을수록 학습 데이터를 완벽히 외울 수 있지만, 노이즈까지 학습하여 새 데이터에서 성능이 급락합니다.

오캄의 면도날: 정규화는 가중치를 작게 유지하여 모델의 복잡도를 제한합니다. "같은 성능이면 더 단순한 모델이 낫다"는 원칙입니다.

🐍 scikit-learn 코드 
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import Ridge, Lasso

poly = PolynomialFeatures(degree=3)
X_poly = poly.fit_transform(X)

model = Ridge(alpha=1.0)  # L2 정규화
# model = Lasso(alpha=1.0)  # L1 정규화
model.fit(X_poly, y)

결정 트리 (Decision Tree)

데이터를 결정 트리로 분류합니다. 각 노드에서 최적의 분할 조건을 찾아 Gini 불순도를 최소화합니다.

💡 왜 결정 트리인가?

해석 가능성: "나이 > 30이고 소득 > 5000이면 승인"처럼 if-else 규칙으로 읽을 수 있어 비전문가도 이해할 수 있습니다.

전처리 불필요: 스케일링, 정규화 없이 원시 데이터를 그대로 사용할 수 있습니다. 범주형/수치형 혼합 데이터도 처리합니다.

🐍 scikit-learn 코드 
from sklearn.tree import DecisionTreeClassifier

tree = DecisionTreeClassifier(
    max_depth=3, criterion='gini'
)
tree.fit(X_train, y_train)
accuracy = tree.score(X_test, y_test)

앙상블 (Ensemble Methods)

여러 약한 모델을 결합하여 강한 모델을 만드는 앙상블 기법입니다. 배깅과 부스팅을 비교합니다.

💡 왜 앙상블인가?

다양성의 힘: 각 모델이 서로 다른 실수를 저지르면, 다수결로 개별 모델보다 더 정확한 결과를 얻습니다.

실전 최강: Kaggle 대회에서 거의 모든 우승 모델이 앙상블을 사용합니다. Random Forest와 XGBoost는 정형 데이터의 표준입니다.

🐍 scikit-learn 코드 
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(
    n_estimators=5, max_depth=3
)
rf.fit(X_train, y_train)
accuracy = rf.score(X_test, y_test)

클러스터링 (Clustering)

레이블 없이 데이터의 자연스러운 그룹을 찾는 비지도 학습입니다. K-Means의 반복 과정을 단계별로 시각화합니다.

💡 왜 클러스터링인가?

레이블이 없을 때: 고객 세분화, 문서 주제 분류, 이상 탐지 등 정답 레이블이 없는 데이터에서도 의미 있는 구조를 발견할 수 있습니다.

데이터 이해: 클러스터링은 데이터 탐색의 첫 단계로, "어떤 패턴이 있는가?"를 먼저 파악하는 데 유용합니다.

🐍 scikit-learn 코드 
from sklearn.cluster import KMeans

kmeans = KMeans(n_clusters=3, init='k-means++')
kmeans.fit(X)
labels = kmeans.labels_
centroids = kmeans.cluster_centers_

실전 플레이그라운드

다양한 데이터셋과 모델을 자유롭게 조합하여 결정 경계를 확인합니다.

💡 왜 플레이그라운드인가?

직관 형성: 같은 데이터에 서로 다른 모델을 적용해 보면, 각 모델의 강점과 한계를 직관적으로 이해할 수 있습니다.

하이퍼파라미터 감각: k, depth, C 같은 하이퍼파라미터를 조절하며 결정 경계의 변화를 즉시 확인할 수 있습니다.

🐍 scikit-learn 코드 
from sklearn.datasets import make_circles
from sklearn.model_selection import train_test_split

X, y = make_circles(n_samples=200, noise=0.15)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
